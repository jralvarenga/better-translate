# Mission

Better Translate is intentionally focused.

Right now, the project is built around the places where type-safe translations feel the most useful today:

- TypeScript
- Node.js and Bun
- React

That focus is not the final destination. It is the starting point.

## Where we are today

Today, Better Translate is designed for teams that want a translation system that feels native inside a modern TypeScript stack.

That means a strong core for TypeScript, practical server support for Node.js and Bun, and a React layer that makes the same translation model work naturally in real apps.

The goal right now is to make that experience solid, predictable, and pleasant before trying to cover everything at once.

## Where we want to go

The long-term mission is bigger than the current surface area.

Better Translate should be able to support more frameworks over time, not just the ones already covered today.

And eventually, the vision goes beyond TypeScript too. The package starts in the TypeScript world because that is where the current strengths are, but the broader goal is to make the ideas behind Better Translate useful in more environments and even in languages outside of TypeScript.

## Why that matters

Translations usually spread across the whole product surface. Because of that, the tooling should be able to grow with the product too.

The mission is to start with a strong, reliable foundation in TypeScript-first apps, then expand carefully without losing the simplicity and safety that made the package worth using in the first place.
