# Next.js

`@better-translate/nextjs` is
the Next.js App Router layer on top of the core package. It adds locale routing config,
proxy helpers, request-scoped server helpers, and locale-aware navigation wrappers.

## When to use it

Use the Next.js adapter when you are using Next.js App Router, you want locale-prefixed
routes, you want request helpers for server components, or you want locale-aware links
and router calls.

## Five-file pattern

A standard Next.js setup needs five files inside your `lib/i18n/` folder:

## routing.ts

```tsx showLineNumbers
import { defineRouting } from "@better-translate/nextjs"

export const routing = defineRouting({
  locales: ["en", "es"] as const,
  defaultLocale: "en",
  routeTemplate: "/[lang]", // scoped: "/app/[lang]"
})
```

## request.ts

```tsx showLineNumbers
import { getRequestConfig } from "@better-translate/nextjs"
import { configureTranslations } from "better-translate"
import { routing } from "./routing"
import { en } from "./messages/en"
import { es } from "./messages/es"

export default getRequestConfig(async ({ locale }) => {
  const translator = await configureTranslations({ en, es })
  return { locale, translator }
})
```

## server.ts

```tsx showLineNumbers
import { createServerHelpers } from "@better-translate/nextjs"
import requestConfig from "./request"

export const {
  getLocale,
  getMessages,
  getTranslations,
  getTranslator,
} = createServerHelpers(requestConfig)
```

## navigation.ts

```tsx showLineNumbers
import { createNavigationFunctions } from "@better-translate/nextjs"
import { routing } from "./routing"

export const {
  I18nLink,
  getPathname,
  useI18nPathname,
  useI18nRouter,
} = createNavigationFunctions({ routing })
```

## Using in server components

```tsx showLineNumbers
import { getTranslations } from "@/lib/i18n/server"

export default async function Page({ params }) {
  const { lang } = await params
  const t = await getTranslations({ locale: lang })

  return <h1>{t("hero.title")}</h1>
}
```

## Root vs scoped routing

With `routeTemplate: "/[lang]"`,
the whole app lives under `/en`, `/es`, etc.

With `routeTemplate: "/app/[lang]"`,
only that subtree is localized. Routes like `/` and `/login` stay outside localization.
