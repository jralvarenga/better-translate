# Skills

The `skills/` folder in
the repository is a learning guide written in plain language. It explains how the library
works without requiring you to read the full source code first.

## Reading order

If you are new, read the skill files in this order:

1. `skills/core/README.md` — The translation engine: typed keys, params, fallback, async loading, and global helpers.
2. `skills/react/README.md` — The React adapter: BetterTranslateProvider and useTranslations().
3. `skills/nextjs/README.md` — The Next.js adapter: locale routing, server helpers, and navigation wrappers.
4. `skills/combined/README.md` — How core, React, and Next.js work together in a real app.
5. `skills/examples/README.md` — A map of the example apps — copy the one that matches your architecture.
6. `skills/workspace/README.md` — How the monorepo is organized: packages vs apps, build scripts, and config files.

## Simple mental model

Better Translate has one main idea: **configure translations once, use typed helpers everywhere.**

The core package owns the translation logic.

The React package makes that logic easy to use in client components.

The Next.js package makes that logic easy to use with routing, server helpers, and locale-aware navigation.

## Which guide should I read?

- → Only need translations in TypeScript or on a server? Read `skills/core/README.md`
- → Building a React app? Read `skills/react/README.md`
- → Building a Next.js App Router app? Read `skills/nextjs/README.md`
- → App has both server and client parts? Read `skills/combined/README.md`
- → Learn best by copying working projects? Read `skills/examples/README.md`
