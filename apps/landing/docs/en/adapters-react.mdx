# React

`@better-translate/react` is
the React layer on top of the core package. It provides
`BetterTranslateProvider` and
`useTranslations()` for
client-side locale switching and context-based translation access across
web React and Expo/React Native apps.

## When to use it

Use the React adapter when your app has client components, you want to switch locale
at runtime, or you want React context instead of calling global helpers directly.

## Expo and React Native

Expo uses this same package. There is no separate native adapter.
If you are wiring Better Translate into an Expo app, keep this provider model
and follow the native setup guide:

## Setup

Create a core translator first, then pass it to `BetterTranslateProvider`.

```tsx showLineNumbers
// i18n.ts
import { configureTranslations } from "better-translate"

export async function createTranslator() {
  return configureTranslations({
    availableLocales: ["en", "es"] as const,
    defaultLocale: "en",
    directions: {
      es: "rtl",
    },
    messages: { en, es },
  })
}
```

```tsx showLineNumbers
// main.tsx
import { BetterTranslateProvider } from "@better-translate/react"
import { createTranslator } from "./i18n"

const translator = await createTranslator()

root.render(
  <BetterTranslateProvider translator={translator}>
    <App />
  </BetterTranslateProvider>
)
```

## useTranslations()

Inside the provider, `useTranslations()` gives
you everything you need:

```tsx showLineNumbers
import { useTranslations } from "@better-translate/react"

function MyComponent() {
  const {
    t,
    locale,
    direction,
    rtl,
    setLocale,
    loadLocale,
    messages,
    supportedLocales,
    defaultLocale,
    fallbackLocale,
    isLoadingLocale,
    loadingLocale,
    localeError,
    translator,
  } = useTranslations()

  return (
    <div>
      <p>{t("greeting", { params: { name: "World" } })}</p>
      <p>{direction}</p>
      <p>{String(rtl)}</p>
      <button type="button" onClick={() => setLocale("es")}>
        Switch to Spanish
      </button>
    </div>
  )
}
```

## Locale switching

`setLocale()` changes
the active locale for that provider tree. If the locale is not cached, it loads it first.
If loading fails, the old locale stays active and `localeError` is set.

`loadLocale()` warms
the cache without changing the active locale. Use it to preload the next locale before
the user switches, so the switch is instant.

```tsx showLineNumbers
// warm the cache before the user switches
await loadLocale("fr")

// switch instantly since it's already cached
setLocale("fr")
```

Use `translator.getDirection()` or `translator.isRtl()` when you need per-call
direction metadata or want to override it through `config.rtl`.

## Provider scope

Each provider has its own active locale state. You can nest multiple providers to give
different parts of the UI independent locale states — useful for testing locale error
behavior in isolation.
