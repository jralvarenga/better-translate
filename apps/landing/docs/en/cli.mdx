# CLI

Use `@better-translate/cli` when you want Better Translate to generate locale files for you.

You do not need the CLI to use the runtime packages. It is optional.

## 1. Install the package

```sh
npm install -D @better-translate/cli
```

## 2. Create a source locale file

Create `src/messages/en.json`:

```json
{
  "home": {
    "title": "Hello",
    "description": "Welcome to the app"
  }
}
```

## 3. Create the CLI config

Create `better-translate.config.ts`:

```ts
import { defineConfig, openai } from "@better-translate/cli/config";

export default defineConfig({
  sourceLocale: "en",
  locales: ["es", "fr"],
  model: openai("gpt-4.1-mini", {
    apiKey: process.env.OPENAI_API_KEY!,
  }),
  messages: {
    entry: "./src/messages/en.json",
  },
});
```

## 4. Run the generator

```sh
npx bt generate --config ./better-translate.config.ts
```

That command creates the target locale files next to your source file.

## 5. Use the generated files in your app

After the files exist, import them into your `@better-translate/core` config just like any hand-written locale file.

## Markdown

If you also want localized markdown generation, add the `markdown.rootDir` option:

```ts
import { defineConfig, openai } from "@better-translate/cli/config";

export default defineConfig({
  sourceLocale: "en",
  locales: ["es", "fr"],
  model: openai("gpt-4.1-mini", {
    apiKey: process.env.OPENAI_API_KEY!,
  }),
  messages: {
    entry: "./src/messages/en.json",
  },
  markdown: {
    rootDir: "./content/docs",
  },
});
```
