# Astro

`@better-translate/astro` is
the Astro layer on top of the core package. It adds request-scoped server helpers,
Astro middleware integration, and localized content-collection helpers for both
`.md` and `.mdx`.

## When to use it

Use the Astro adapter when you are building an Astro app, want request-scoped
translation helpers that follow the current locale, and want one localized content
helper that works with Astro content collections for Markdown and MDX.

## Mental model

Astro keeps ownership of routing.

- Astro i18n config stays in `astro.config.mjs`
- Better Translate config stays in `src/lib/i18n/request.ts`
- Astro middleware stores the resolved request locale for Better Translate
- Astro content collections still render the content

A standard Astro setup needs four files:

## request.ts

```ts showLineNumbers
import { getRequestConfig } from "@better-translate/astro"
import { configureTranslations } from "better-translate/core"

import { en } from "./messages/en"
import { es } from "./messages/es"

export const requestConfig = getRequestConfig(async () => ({
  translator: await configureTranslations({
    availableLocales: ["en", "es"] as const,
    defaultLocale: "en",
    fallbackLocale: "en",
    messages: {
      en,
      es,
    },
  }),
}))
```

## server.ts

```ts showLineNumbers
import {
  createServerHelpers,
  setRequestLocale,
} from "@better-translate/astro"

import { requestConfig } from "./request"

export const {
  getAvailableLanguages,
  getDirection,
  getLocale,
  getMessages,
  getTranslations,
  getTranslator,
  isRtl,
} = createServerHelpers(requestConfig)

export { setRequestLocale }
```

## middleware.ts

```ts showLineNumbers
import { createBetterTranslateMiddleware } from "@better-translate/astro/middleware"

import { requestConfig } from "./lib/i18n/request"

export const onRequest = createBetterTranslateMiddleware(requestConfig)
```

Astro’s built-in i18n remains the routing source of truth. The adapter just reads the
resolved locale from the Astro request lifecycle and stores it for Better Translate.

## content.ts

```ts showLineNumbers
import { getCollection, render } from "astro:content"
import { createContentCollectionHelpers } from "@better-translate/astro/content"

import { requestConfig } from "./request"

export const docs = createContentCollectionHelpers(requestConfig, {
  collection: "docs",
  getCollection,
  render,
})
```

Use locale-prefixed entry ids in the collection:

```txt
src/content/docs/
  en/getting-started.md
  en/component-demo.mdx
  es/getting-started.md
```

The helper resolves the requested locale first, then the translator fallback locale.
Because Astro’s `render(entry)` handles the final rendering, the same helper works for
both `.md` and `.mdx`.

## Using in `.astro` pages

```astro showLineNumbers
---
import { docs } from "@/src/lib/i18n/content"
import { getLocale, getTranslations } from "@/src/lib/i18n/server"

const locale = await getLocale()
const t = await getTranslations()
const document = await docs.renderDocument("getting-started", { locale })
const { Content } = document.rendered
---

<h1>{t("docs.title")}</h1>
{document.usedFallback && <p>Fallback content</p>}
<Content />
```
