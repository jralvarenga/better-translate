# واجهة سطر الأوامر

يأتي Better Translate مع سكريبتات مساحة العمل للبناء وفحص الأنواع والتطوير
للحزم والتطبيقات في المستودع الموحد. إذا كنت تستخدم هذا كمكتبة في مشروعك الخاص،
فهذه هي السكريبتات التي ستستخدمها أكثر.

## سكريبتات مساحة العمل

شغّل هذه من جذر المستودع باستخدام Bun وTurborepo.

- `bun run dev` ,  تشغيل جميع التطبيقات في وضع التطوير.
- `bun run build` ,  بناء جميع الحزم والتطبيقات.
- `bun run build:packages` ,  بناء حزم المكتبة فقط.
- `bun run check-types` ,  تشغيل فحص أنواع TypeScript عبر مساحة العمل بالكامل.
- `bun run check-types:packages` ,  تشغيل فحص الأنواع على الحزم فقط.

## توليد مخطط JSON

تُصدّر الحزمة الأساسية `createTranslationJsonSchema` لتوليد
مخطط JSON من كائن اللغة المصدر. يساعد هذا المحررين في التحقق من صحة ملفات
اللغة الشقيقة.

```tsx showLineNumbers
import { createTranslationJsonSchema } from "better-translate"
import { en } from "./messages/en"
import { writeFileSync } from "fs"

const schema = createTranslationJsonSchema(en)
writeFileSync("./messages/schema.json", JSON.stringify(schema, null, 2))
```

## سكريبتات كل حزمة

كل حزمة لها سكريبتاتها المحلية الخاصة. من أي دليل حزمة:

```tsx showLineNumbers
# بناء الحزمة
bun run build

# فحص أنواع الحزمة
bun run check-types

# تشغيل الاختبارات (حيثما توفرت)
bun test
```
