# واجهة سطر الأوامر

استخدم `@better-translate/cli` عندما تريد من Better Translate إنشاء ملفات اللغات وتحديثها لك.

أنت لا تحتاج إلى CLI لاستخدام حزم وقت التشغيل. هي اختيارية.

## 1. تثبيت الحزم

```sh
npm install -D @better-translate/cli @ai-sdk/openai
```

أو ثبّت حزمة موفّر أخرى:

```sh
npm install -D @better-translate/cli @ai-sdk/anthropic
# أو
npm install -D @better-translate/cli @ai-sdk/moonshotai
```

لم تعد CLI تتضمن مساعدات خاصة بالموفّر. تطبيقك هو الذي يثبّت موفّر AI SDK ويضبطه مباشرة.

## 2. أنشئ ملف اللغة المصدر

أنشئ `src/messages/en.json`:

```json
{
  "home": {
    "title": "Hello",
    "description": "Welcome to the app"
  }
}
```

## 3. أنشئ إعداد CLI

أنشئ `better-translate.config.ts`. مسار `model` المباشر هو نفسه مع أي موفّر:

```ts
import { openai } from "@ai-sdk/openai";
import { defineConfig } from "@better-translate/cli/config";

export default defineConfig({
  sourceLocale: "en",
  locales: ["es", "fr"],
  model: openai("gpt-5"),
  messages: {
    entry: "./src/messages/en.json",
  },
});
```

```ts
import { anthropic } from "@ai-sdk/anthropic";
import { defineConfig } from "@better-translate/cli/config";

export default defineConfig({
  sourceLocale: "en",
  locales: ["es", "fr"],
  model: anthropic("claude-sonnet-4-5"),
  messages: {
    entry: "./src/messages/en.json",
  },
});
```

```ts
import { moonshotai } from "@ai-sdk/moonshotai";
import { defineConfig } from "@better-translate/cli/config";

export default defineConfig({
  sourceLocale: "en",
  locales: ["es", "fr"],
  model: moonshotai("kimi-k2-0905-preview"),
  messages: {
    entry: "./src/messages/en.json",
  },
});
```

إذا كنت تحتاج إعدادات مخصّصة للموفّر، فأنشئ النموذج بنفسك ومرّره إلى CLI. بيانات الاعتماد والإعدادات الخاصة بالموفّر تبقى داخل حزمة الموفّر التي تختارها:

```ts
import { createOpenAI } from "@ai-sdk/openai";
import { defineConfig } from "@better-translate/cli/config";

const model = createOpenAI({
  apiKey: process.env.OPENAI_API_KEY,
  baseURL: process.env.OPENAI_BASE_URL,
})("gpt-5");

export default defineConfig({
  sourceLocale: "en",
  locales: ["es", "fr"],
  model,
  messages: {
    entry: "./src/messages/en.json",
  },
});
```

هذا تغيير غير متوافق مع الإصدارات الأقدم من CLI التي كانت تصدّر `openai(...)` من `@better-translate/cli/config`.

## 4. علّم السلاسل المعلّقة

علّم السلاسل باستخدام `{ bt: true }` لكي تكتب النص المصدر مباشرة، وستقوم CLI بتسمية المفاتيح نيابةً عنك:

```ts
import { t } from "@better-translate/core";

export function navLabel() {
  return t("Home", { bt: true });
}
```

الخيار `{ bt: true }` يجعل الوسيط الأول أكثر مرونة حتى تتمكن من كتابة النص المصدر مباشرة. أثناء التشغيل يعيد النص كما هو إلى أن تتم عملية الاستخراج.

## 5. استخرج المفاتيح المصدرية

```sh
npx bt extract --config ./better-translate.config.ts
```

هذا الأمر يفحص الاستدعاءات المعلّمة `t(..., { bt: true })` ويضيف المفاتيح الناقصة إلى ملف اللغة المصدر ويعيد كتابة الاستدعاءات الناجحة إلى `t("namespace.generatedKey")`.

## 6. شغّل المولّد

```sh
npx bt generate --config ./better-translate.config.ts
```

هذا الأمر ينشئ ملفات اللغات الهدف بجانب ملف اللغة المصدر.

إذا كان `markdown.rootDir` مفعّلًا وكانت العملية ستنشئ أو تستبدل ملفات `.md` أو `.mdx` المترجمة، فستطلب CLI تأكيدًا قبل إجراء التغييرات. استخدم `--yes` أو `-y` لتجاوز التأكيد:

```sh
npx bt generate --yes
```

في البيئات غير التفاعلية، يجب تمرير `--yes` عندما تحتاج العملية إلى كتابة ملفات Markdown مترجمة.

## 7. استخدم الملفات المولّدة في تطبيقك

بعد إنشاء الملفات، استوردها في إعداد `@better-translate/core` تماماً كما تفعل مع أي ملفات لغات مكتوبة يدوياً.

## Markdown

إذا كنت تريد أيضاً توليد ملفات Markdown مترجمة، فأضف الخيار `markdown.rootDir`. هذا يعمل بالطريقة نفسها مع أي نموذج موفّر من AI SDK:

```ts
import { openai } from "@ai-sdk/openai";
import { defineConfig } from "@better-translate/cli/config";

export default defineConfig({
  sourceLocale: "en",
  locales: ["es", "fr"],
  model: openai("gpt-5"),
  messages: {
    entry: "./src/messages/en.json",
  },
  markdown: {
    rootDir: "./content/docs",
  },
});
```
