# RTL

يحتوي `@better-translate/core` على دعم مدمج للغات من اليمين لليسار.
تُعلن في الإعدادات عن اللغات التي تُكتب من اليمين لليسار، ثم تستعلم عن الاتجاه في وقت التشغيل
باستخدام `getDirection` أو `isRtl`.

## متى تستخدمه

استخدم دعم الاتجاه من اليمين لليسار عندما يحتاج تطبيقك إلى دعم لغات مثل العربية، العبرية، الفارسية، أو الأردوية.
توفر لك دوال الاتجاه طريقة مكتوبة لتعيين سمة `dir` على عناصر HTML
من دون إدارة جدول بحث منفصل.

## إعلان الاتجاهات

أضف مفتاح `directions` إلى استدعاء `configureTranslations`. أي لغة تُحذف تُعين افتراضيًا إلى `"ltr"`.

```tsx showLineNumbers
import { configureTranslations } from "@better-translate/core";

const translator = await configureTranslations({
  availableLocales: ["en", "ar"] as const,
  defaultLocale: "en",
  messages: { en, ar },
  directions: {
    ar: "rtl",
  },
});
```

## الاستعلام عن الاتجاه

استدعِ `getDirection` للحصول على `"ltr"` أو `"rtl"` لأي لغة.
استدعِ `isRtl` للحصول على اختصار منطقي.

```tsx showLineNumbers
translator.getDirection({ locale: "ar" }); // "rtl"
translator.getDirection({ locale: "en" }); // "ltr"

translator.isRtl({ locale: "ar" }); // true
translator.isRtl({ locale: "en" }); // false
```

## التجاوز لكل استدعاء

مرر `config.rtl` لتجاوز الاتجاه المُعين لاستدعاء واحد.
هذا لا يؤثر على البحث عن الرسالة، بل يغير فقط الاتجاه الذي يُحل.

```tsx showLineNumbers
// إجبار LTR حتى للغة RTL
translator.t("greeting", {
  locale: "ar",
  config: { rtl: false },
});

// إجبار RTL حتى للغة LTR
translator.t("greeting", {
  locale: "en",
  config: { rtl: true },
});
```

## التطبيق على HTML

استخدم `getDirection` عند عرض سمة `dir` على العناصر الجذرية.

```tsx showLineNumbers
// على عنصر html (مثلاً في تخطيط Next.js)
<html lang={locale} dir={translator.getDirection({ locale })}>

// على حاوية محددة
<div dir={translator.getDirection({ locale })}>
  {t("greeting")}
</div>
```
