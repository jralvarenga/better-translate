# Core

`@better-translate/core` يعمل في أي مشروع TypeScript. لا يوجد اعتماد على إطار عمل، ولا يوجد متطلب وقت تشغيل. نفس الإعدادات ونفس واجهة البرمجة تنطبق سواء كنت في Next.js أو Astro أو React أو Bun أو Node.js أو سكربت أو أي بيئة TypeScript أخرى.

**المحولات** هي امتدادات للنواة تضيف مساعدات خاصة بالإطار: توجيه يعتمد على اللغة، سياق React، مساعدات لكل طلب، إلخ. لكن كل شيء مبني على نفس النواة.

## متى تستخدم النواة مباشرة

استخدم النواة بدون محول عندما لا تحتاج إلى مساعدات خاصة بالإطار. يشمل ذلك الخوادم، وواجهات برمجة التطبيقات، والسكربتات، والمكتبات المشتركة، أو أي بيئة TypeScript حيث لا يوجد محول لإعدادك.

## 1. تثبيت الحزمة

```sh
npm install @better-translate/core
```

## 2. إنشاء ملف مترجم واحد

أنشئ `src/i18n.ts`:

```ts
import { configureTranslations } from "@better-translate/core";

const en = {
  home: {
    title: "Hello",
    description: "This is the English version.",
  },
} as const;

const es = {
  home: {
    title: "Hola",
    description: "Esta es la version en espanol.",
  },
} as const;

export const translator = await configureTranslations({
  availableLocales: ["en", "es"] as const,
  defaultLocale: "en",
  fallbackLocale: "en",
  messages: { en, es },
});
```

## 3. استخدام المترجم في أي مكان

```ts
import { translator } from "./i18n";

translator.t("home.title"); // "Hello"
translator.t("home.title", { locale: "es" }); // "Hola"
translator.t("home.description", { locale: "es" });
```

## 4. تحديث `src/i18n.ts` لإضافة لغة أخرى لاحقاً

```ts
const fr = {
  home: {
    title: "Bonjour",
    description: "Ceci est la version francaise.",
  },
} as const;

// استبدل تعريف translator السابق بهذا التعريف المحدّث
export const translator = await configureTranslations({
  availableLocales: ["en", "es", "fr"] as const,
  defaultLocale: "en",
  fallbackLocale: "en",
  messages: { en, es, fr },
});
```

## سلوك اللغة الاحتياطية

إذا لم يكن للغة النشطة مفتاح معين، يحاول better-translate اللغة الاحتياطية.
إذا لم تكن اللغة الاحتياطية تحتوي عليه أيضاً، يُرجع سلسلة المفتاح نفسها.

- → قيمة اللغة مفقودة → قيمة اللغة الاحتياطية
- → قيمة اللغة الاحتياطية مفقودة → سلسلة المفتاح

## محمّلات غير متزامنة

سجّل محمّلات اللغات للغات لا تريد تحميلها مسبقاً. يتم تخزين اللغات المحمّلة مؤقتاً بعد أول تحميل ناجح.

```ts
const translator = await configureTranslations({
  availableLocales: ["en", "fr"] as const,
  defaultLocale: "en",
  messages: { en },
  loaders: {
    fr: async () => import("./messages/fr").then((m) => m.default),
  },
});

// يحمّل fr عند الطلب ويخزنه مؤقتاً
await translator.loadLocale("fr");
```

## 5. استخراج السلاسل تلقائياً باستخدام CLI

بدلاً من تسمية المفاتيح يدوياً، علّم السلاسل بـ `{ bt: true }` ودع CLI يستخرجها ويسمّيها تلقائياً:

```ts
import { t } from "@better-translate/core";

// اكتب السلسلة المصدر مباشرة — CLI يحوّلها إلى مفتاح صحيح
t("Hello world", { bt: true });
```

شغّل `npx bt extract` لمزامنة السلاسل في ملف اللغة المصدر وإعادة كتابة الاستدعاءات إلى مفاتيح صحيحة. راجع [وثائق CLI](/docs/cli) للإعداد الكامل.
