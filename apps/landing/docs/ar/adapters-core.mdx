# النواة

تعمل `@better-translate/core` في أي مشروع TypeScript. لا اعتماد على إطار بعينه، ولا اشتراط لبيئة تشغيل محددة. نفس الإعداد ونفس الـ API ينطبقان سواء كنت في Next.js أو Astro أو React أو Bun أو Node.js أو سكريبت أو أي بيئة TypeScript أخرى.

**المحولات** هي امتدادات للنواة تضيف مساعدات خاصة بالإطار: توجيه مدرك للغة، وcontext الخاصة بـ React، ومساعدات لكل طلب، وغيرها. لكن كل شيء مبني فوق نفس النواة.

## متى تستخدم النواة مباشرة

استخدم النواة بدون محول عندما لا تحتاج إلى مساعدات خاصة بإطار عمل. يشمل ذلك الخوادم وAPIs والسكريبتات والمكتبات المشتركة أو أي بيئة TypeScript لا يوجد لها محول مخصص.

## إعداد النموذج المختصر

مرّر خريطة اللغة مباشرة. تصبح اللغة الأولى الافتراضية والاحتياطية.

```tsx showLineNumbers
import { configureTranslations } from "@better-translate/core"

const en = { greeting: "Hello, {name}!" }
const ar = { greeting: "مرحباً، {name}!" }

const translator = await configureTranslations({ en, ar })

const t = translator.getTranslator("ar")
t("greeting", { params: { name: "عالم" } }) // "مرحباً، عالم!"
```

## إعداد النموذج بالخيارات

حدد عقد اللغة مسبقاً. يدعم هذا أدوات التحميل غير المتزامن وهو الإعداد الموصى به للتطبيقات الإنتاجية.

```tsx showLineNumbers
import { configureTranslations } from "@better-translate/core"

const translator = await configureTranslations({
  availableLocales: ["en", "ar", "fr"] as const,
  defaultLocale: "en",
  fallbackLocale: "en",
  directions: {
    ar: "rtl",
  },
  messages: { en, ar },
  loaders: {
    fr: async () => ({
      greeting: "Bonjour, {name}!",
    }),
  },
})
```

`directions` اختياري. أي لغة تحذفها تكون افتراضياً `"ltr"`.

## المساعدات العالمية

بعد الضبط مرة واحدة، يمكنك استدعاء هذه المساعدات العالمية من أي مكان:

```tsx showLineNumbers
import { t, loadLocale, getSupportedLocales, getMessages } from "@better-translate/core"

// ترجمة باللغة النشطة
t("greeting", { params: { name: "عالم" } })

// تحميل لغة في ذاكرة التخزين المؤقت بدون التبديل
await loadLocale("fr")

// الحصول على جميع رموز اللغة المدعومة
getSupportedLocales() // ["en", "ar", "fr"]

// الحصول على لقطة مجمّدة من الرسائل المحملة
getMessages()
```

## سلوك الاحتياطية

إذا لم تمتلك اللغة النشطة مفتاحاً، يحاول better-translate اللغة الاحتياطية.
إذا لم تمتلك الاحتياطية أيضاً المفتاح، يُعيد سلسلة المفتاح نفسها.

- → قيمة اللغة مفقودة → قيمة اللغة الاحتياطية
- → قيمة اللغة الاحتياطية مفقودة → سلسلة المفتاح

## أدوات التحميل غير المتزامن

سجّل أدوات تحميل اللغة للغات التي لا تريد تحميلها مسبقاً. اللغات المحملة تُخزّن في الذاكرة المؤقتة بعد أول تحميل ناجح.

```tsx showLineNumbers
const translator = await configureTranslations({
  availableLocales: ["en", "fr"] as const,
  defaultLocale: "en",
  messages: { en },
  loaders: {
    fr: async () => import("./messages/fr").then((m) => m.default),
  },
})

// يحمّل fr عند الطلب ويخزنه في الذاكرة المؤقتة
await translator.loadLocale("fr")
```

## بيانات الاتجاه

استخدم المترجم المُعدَّ عندما تحتاج بيانات اتجاه اللغة:

```tsx showLineNumbers
translator.getDirection({ locale: "ar" }) // "rtl"
translator.isRtl({ locale: "ar" }) // true

// تجاوزات rtl لكل استدعاء لا تغير بحث الرسائل.
translator.t("greeting", {
  locale: "ar",
  config: {
    rtl: false,
  },
})
```
