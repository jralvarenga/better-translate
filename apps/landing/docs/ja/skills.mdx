# Skills

`skills/` フォルダは、コードベース全体を読む前にパッケージを正しく統合する方法を説明します。

以下のような質問に対してシンプルな答えが欲しい時に使用してください：

- どのパッケージから始めればいいですか？
- Reactも必要ですか、それともNext.jsだけでいいですか？
- MDXはセットアップにどう組み込まれますか？
- アプリに最適な最小限のパッケージの組み合わせは何ですか？

## 推奨の読み順

1. `skills/README.md` — パッケージマップ
2. `skills/core/README.md` — 翻訳機能を正しく起動する
3. `skills/cli/README.md` — CLIでロケールファイルを生成したり、`bt`でマークされた文字列を抽出したい場合
4. `skills/react/README.md` — UIがReactまたはExpoの場合
5. `skills/nextjs/README.md` — アプリがNext.js App Routerの場合
6. `skills/md/README.md` — コンテンツが `.md` または `.mdx` にある場合
7. `skills/combined/README.md` — アプリがルーティング、サーバーレンダリング、クライアントフックを混在させている場合

## パッケージの簡易選択

- プレーンなTypeScript、Node.js、API、または共有ライブラリ: `@better-translate/core`
- React WebまたはExpo: `@better-translate/core` + `@better-translate/react`
- Next.js App Router: `@better-translate/core` + `@better-translate/nextjs`
- クライアントサイドフックを使うNext.js: `@better-translate/react` を追加
- TanStack Router: `@better-translate/core` + `@better-translate/tanstack-router`、通常 `@better-translate/react` も
- Astro: `@better-translate/core` + `@better-translate/astro`
- ローカライズされたMarkdownまたはMDX: `@better-translate/md` を追加
- AI生成のロケールファイルまたは自動抽出された文字列: `@better-translate/cli` を追加
