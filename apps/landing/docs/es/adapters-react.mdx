# React

Usa `@better-translate/react` cuando los componentes de React necesiten traducciones y cambio de idioma.

## 1. Instala los paquetes

```sh
npm install @better-translate/core @better-translate/react
```

## 2. Crea el traductor compartido

Crea `src/i18n.ts`:

```ts
import { configureTranslations } from "@better-translate/core";

const en = {
  home: {
    title: "Hello",
  },
} as const;

const es = {
  home: {
    title: "Hola",
  },
} as const;

export const translator = await configureTranslations({
  availableLocales: ["en", "es"] as const,
  defaultLocale: "en",
  fallbackLocale: "en",
  messages: { en, es },
});
```

## 3. Envuelve tu aplicación

Crea o actualiza `src/main.tsx`:

```tsx
import React from "react";
import ReactDOM from "react-dom/client";

import { BetterTranslateProvider } from "@better-translate/react";

import { App } from "./app";
import { translator } from "./i18n";

ReactDOM.createRoot(document.getElementById("root")!).render(
  <React.StrictMode>
    <BetterTranslateProvider translator={translator}>
      <App />
    </BetterTranslateProvider>
  </React.StrictMode>,
);
```

## 4. Lee las traducciones dentro de un componente

Crea `src/header.tsx`:

```tsx
import { useTranslations } from "@better-translate/react";

import { translator } from "./i18n";

export function Header() {
  const { locale, setLocale, t } = useTranslations<typeof translator>();

  return (
    <header>
      <h1>{t("home.title")}</h1>
      <button onClick={() => setLocale("en")} disabled={locale === "en"}>
        English
      </button>
      <button onClick={() => setLocale("es")} disabled={locale === "es"}>
        Espanol
      </button>
    </header>
  );
}
```

## Opcional: tipa `useTranslations()` una vez para toda la aplicación

Si quieres que `useTranslations()` autocomplete las claves de traducción sin
repetir `<typeof translator>` en cada componente, añade un archivo de declaración:

Crea `src/better-translate.d.ts`:

```ts
import { translator } from "./i18n";

declare module "@better-translate/react" {
  interface BetterTranslateReactTypes {
    translator: typeof translator;
  }
}
```

Entonces tus componentes pueden llamar al hook sin un genérico:

```tsx
import { useTranslations } from "@better-translate/react";

export function Header() {
  const { locale, setLocale, t } = useTranslations();

  return (
    <header>
      <h1>{t("home.title")}</h1>
      <button onClick={() => setLocale("en")} disabled={locale === "en"}>
        English
      </button>
      <button onClick={() => setLocale("es")} disabled={locale === "es"}>
        Espanol
      </button>
    </header>
  );
}
```

Los genéricos explícitos siguen funcionando, así que puedes seguir usando `useTranslations<typeof translator>()`
donde eso se ajuste mejor a tu código.

## Cuándo usar solo React

Usa solo `@better-translate/core` + `@better-translate/react` cuando:

- tu aplicación es una SPA de React
- tu aplicación es una aplicación de Expo
- los cambios de idioma ocurren en componentes de cliente

Si estás en Next.js App Router, conserva este paquete para los hooks de cliente y añade el adaptador de Next.js para el enrutamiento y helpers del servidor.

## Genera archivos de idioma automáticamente

En lugar de escribir cada traducción a mano, usa la CLI para extraer cadenas y generar archivos de idioma: [Guía de CLI](/docs/cli)
