# React

`@better-translate/react` es
la capa React sobre el paquete core. Provee
`BetterTranslateProvider` y
`useTranslations()` para
cambio de locale en el cliente y acceso a traducciones basado en context
en apps web React y Expo/React Native.

## Cuándo usarlo

Usa el adaptador React cuando tu app tenga componentes cliente, quieras cambiar el locale
en tiempo de ejecución, o prefieras React context en lugar de llamar directamente a los helpers globales.

## Expo y React Native

Expo usa este mismo paquete. No existe un adaptador nativo separado.
Si vas a integrar Better Translate en una app Expo, conserva este modelo de provider
y sigue la guía dedicada:

## Configuración

Crea primero un translator del core, luego pásalo a `BetterTranslateProvider`.

```tsx showLineNumbers
// i18n.ts
import { configureTranslations } from "better-translate"

export async function createTranslator() {
  return configureTranslations({
    availableLocales: ["en", "es"] as const,
    defaultLocale: "en",
    directions: {
      es: "rtl",
    },
    messages: { en, es },
  })
}
```

```tsx showLineNumbers
// main.tsx
import { BetterTranslateProvider } from "@better-translate/react"
import { createTranslator } from "./i18n"

const translator = await createTranslator()

root.render(
  <BetterTranslateProvider translator={translator}>
    <App />
  </BetterTranslateProvider>
)
```

## useTranslations()

Dentro del provider, `useTranslations()` te da
todo lo que necesitas:

```tsx showLineNumbers
import { useTranslations } from "@better-translate/react"

function MiComponente() {
  const {
    t,
    locale,
    direction,
    rtl,
    setLocale,
    loadLocale,
    messages,
    supportedLocales,
    defaultLocale,
    fallbackLocale,
    isLoadingLocale,
    loadingLocale,
    localeError,
    translator,
  } = useTranslations()

  return (
    <div>
      <p>{t("greeting", { params: { name: "Mundo" } })}</p>
      <p>{direction}</p>
      <p>{String(rtl)}</p>
      <button type="button" onClick={() => setLocale("en")}>
        Cambiar a inglés
      </button>
    </div>
  )
}
```

## Cambio de locale

`setLocale()` cambia
el locale activo para ese árbol de provider. Si el locale no está en caché, lo carga primero.
Si la carga falla, el locale anterior permanece activo y `localeError` se establece.

`loadLocale()` calienta
el caché sin cambiar el locale activo. Úsalo para precargar el próximo locale antes de que
el usuario cambie, para que el cambio sea instantáneo.

```tsx showLineNumbers
// calentar el caché antes de que el usuario cambie
await loadLocale("fr")

// cambiar instantáneamente ya que está en caché
setLocale("fr")
```

Usa `translator.getDirection()` o `translator.isRtl()` cuando necesites
metadatos de dirección por llamada o quieras sobrescribirlos con `config.rtl`.

## Alcance del provider

Cada provider tiene su propio estado de locale activo. Puedes anidar múltiples providers para dar
a diferentes partes de la UI estados de locale independientes ,  útil para probar el comportamiento
de error de locale en aislamiento.
