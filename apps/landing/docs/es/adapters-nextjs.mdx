# Next.js

`@better-translate/nextjs` es
la capa Next.js App Router sobre el paquete core. Agrega configuración de enrutamiento por locale,
helpers proxy, helpers de servidor por request y wrappers de navegación con locale.

## Cuándo usarlo

Usa el adaptador Next.js cuando uses Next.js App Router, quieras rutas con prefijo de locale,
helpers de request para server components, o links y llamadas de router con locale.

## Patrón de cinco archivos

Una configuración estándar de Next.js necesita cinco archivos dentro de tu carpeta `lib/i18n/`:

## routing.ts

```tsx showLineNumbers
import { defineRouting } from "@better-translate/nextjs"

export const routing = defineRouting({
  locales: ["en", "es"] as const,
  defaultLocale: "en",
  routeTemplate: "/[lang]", // acotado: "/app/[lang]"
})
```

## request.ts

```tsx showLineNumbers
import { getRequestConfig } from "@better-translate/nextjs/server"
import { configureTranslations } from "better-translate"
import { routing } from "./routing"
import { en } from "./messages/en"
import { es } from "./messages/es"

export default getRequestConfig(async () => {
  const translator = await configureTranslations({ en, es })
  return { translator }
})
```

## server.ts

```tsx showLineNumbers
import { createServerHelpers, setRequestLocale } from "@better-translate/nextjs/server"
import requestConfig from "./request"

export const {
  getLocale,
  getMessages,
  getTranslations,
  getTranslator,
} = createServerHelpers(requestConfig)

export { setRequestLocale }
```

## layout.tsx

```tsx showLineNumbers
import { hasLocale } from "@better-translate/nextjs"
import { setRequestLocale } from "@better-translate/nextjs/server"
import { notFound } from "next/navigation"
import { routing } from "./routing"

export default async function Layout({ children, params }) {
  const { lang } = await params

  if (!hasLocale(routing.locales, lang)) {
    notFound()
  }

  setRequestLocale(lang)

  return children
}
```

## navigation.ts

```tsx showLineNumbers
import { createNavigationFunctions } from "@better-translate/nextjs"
import { routing } from "./routing"

export const {
  I18nLink,
  getPathname,
  useI18nPathname,
  useI18nRouter,
} = createNavigationFunctions({ routing })
```

## Uso en server components

```tsx showLineNumbers
import { getTranslations } from "@/lib/i18n/server"

export default async function Page({ params }) {
  await params
  const t = await getTranslations()

  return <h1>{t("hero.title")}</h1>
}
```

## Enrutamiento raíz vs acotado

Con `routeTemplate: "/[lang]"`,
toda la app vive bajo `/en`, `/es`, etc.

Con `routeTemplate: "/app/[lang]"`,
solo ese subárbol está localizado. Rutas como `/` y `/login` quedan fuera de la localización.
