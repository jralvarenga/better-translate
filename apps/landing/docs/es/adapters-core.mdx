# Core

`better-translate` funciona en cualquier proyecto TypeScript. No hay dependencias de framework ni de entorno de ejecución. La misma configuración y la misma API aplican ya sea que estés en Next.js, Astro, React, Bun, Node.js, un script o cualquier otro entorno TypeScript.

Los **adaptadores** son extensiones del core que añaden helpers específicos del framework: rutas con locale, context de React, helpers por request, etc. Pero todo se apoya en el mismo core.

## Cuándo usarlo directamente

Usa el paquete core sin adaptadores cuando no necesites helpers de framework. Esto incluye servidores, APIs, scripts, librerías compartidas o cualquier entorno TypeScript donde no uses un framework con adaptador propio.

## Configuración en forma corta

Pasa un mapa de locales directamente. El primer locale se convierte en el predeterminado y fallback.

```tsx showLineNumbers
import { configureTranslations } from "better-translate"

const en = { greeting: "Hello, {name}!" }
const es = { greeting: "¡Hola, {name}!" }

const translator = await configureTranslations({ en, es })

const t = translator.getTranslator("es")
t("greeting", { params: { name: "Mundo" } }) // "¡Hola, Mundo!"
```

## Configuración en forma de opciones

Define el contrato de locales por adelantado. Esto soporta loaders asíncronos y es la configuración
recomendada para apps en producción.

```tsx showLineNumbers
import { configureTranslations } from "better-translate"

const translator = await configureTranslations({
  availableLocales: ["en", "es", "fr"] as const,
  defaultLocale: "en",
  fallbackLocale: "en",
  directions: {
    es: "rtl",
  },
  messages: { en, es },
  loaders: {
    fr: async () => ({
      greeting: "Bonjour, {name}!",
    }),
  },
})
```

`directions` es opcional. Cualquier locale omitido usa `"ltr"` por defecto.

## Helpers globales

Después de configurar una vez, puedes llamar estos helpers globales desde cualquier lugar:

```tsx showLineNumbers
import { t, loadLocale, getSupportedLocales, getMessages } from "better-translate"

// traducir con el locale activo
t("greeting", { params: { name: "Mundo" } })

// cargar un locale en caché sin cambiar el activo
await loadLocale("fr")

// obtener todos los códigos de locale soportados
getSupportedLocales() // ["en", "es", "fr"]

// obtener un snapshot congelado de los mensajes cargados
getMessages()
```

## Comportamiento de fallback

Si el locale activo no tiene una clave, Better Translate intenta con el locale fallback.
Si el fallback tampoco tiene la clave, devuelve el string de la clave misma.

- → Valor de locale faltante → valor del locale fallback
- → Valor de fallback faltante → string de la clave

## Loaders asíncronos

Registra loaders de locale para idiomas que no quieras precargar. Los locales cargados se
almacenan en caché después de la primera carga exitosa.

```tsx showLineNumbers
const translator = await configureTranslations({
  availableLocales: ["en", "fr"] as const,
  defaultLocale: "en",
  messages: { en },
  loaders: {
    fr: async () => import("./messages/fr").then((m) => m.default),
  },
})

// carga fr bajo demanda y lo almacena en caché
await translator.loadLocale("fr")
```

## Metadatos de dirección

Usa el translator configurado cuando necesites metadatos de dirección por locale:

```tsx showLineNumbers
translator.getDirection({ locale: "es" }) // "rtl"
translator.isRtl({ locale: "es" }) // true

// Los overrides rtl por llamada no cambian la búsqueda del mensaje.
translator.t("greeting", {
  locale: "es",
  config: {
    rtl: false,
  },
})
```

## Generación de esquema JSON

Construye un JSON Schema desde tu locale fuente para ayudar a los editores a validar archivos de locale hermanos.

```tsx showLineNumbers
import { createTranslationJsonSchema } from "better-translate"

const schema = createTranslationJsonSchema(en)
// escribir el esquema en disco para habilitar validación del editor
```
