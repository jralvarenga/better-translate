# TanStack Router

Use `@better-translate/tanstack-router` cuando tu aplicación usa TanStack Router o TanStack Start y quieres URLs consciente de la localización.

En la mayoría de las aplicaciones reales lo combinarás con:

- `@better-translate/core` para el traductor
- `@better-translate/react` para los hooks dentro de los componentes React

## 1. Instala los paquetes

```sh
npm install @better-translate/core @better-translate/react @better-translate/tanstack-router
```

## 2. Crea el traductor compartido

Crea `src/lib/i18n.ts`:

```ts
import { configureTranslations } from "@better-translate/core";

const en = {
  home: {
    title: "Hello from TanStack Router",
  },
} as const;

const es = {
  home: {
    title: "Hola desde TanStack Router",
  },
} as const;

export const translator = await configureTranslations({
  availableLocales: ["en", "es"] as const,
  defaultLocale: "en",
  fallbackLocale: "en",
  messages: { en, es },
});
```

## 3. Agrega la configuración de enrutamiento de localización

Crea `src/lib/routing.ts`:

```ts
import { defineRouting } from "@better-translate/tanstack-router";

export const routing = defineRouting({
  defaultLocale: "en",
  locales: ["en", "es"] as const,
  routeTemplate: "/$lang",
});
```

## 4. Crea helpers de navegación localizados

Crea `src/lib/navigation.tsx`:

```tsx
import {
  Link,
  useLocation,
  useNavigate,
  useRouter,
} from "@tanstack/react-router";

import { createNavigationFunctions } from "@better-translate/tanstack-router/navigation";

import { routing } from "./routing";

export const {
  Link: I18nLink,
  useLocale,
  useNavigate: useI18nNavigate,
  usePathname: useI18nPathname,
} = createNavigationFunctions({
  Link,
  routing,
  useLocation,
  useNavigate,
  useRouter,
});
```

Usa `routeTemplate: "/$lang"` o `routeTemplate: "/{$lang}"` para segmentos de locale obligatorios. Usa `"/{-$lang}"` cuando el locale por defecto no debe aparecer en la URL.

## 5. Envuelve la aplicación con el proveedor de React

Crea `src/app.tsx`:

```tsx
import { RouterProvider } from "@tanstack/react-router";

import { BetterTranslateProvider } from "@better-translate/react";

import { router } from "./router";
import { translator } from "./lib/i18n";

export function App() {
  return (
    <BetterTranslateProvider translator={translator}>
      <RouterProvider router={router} />
    </BetterTranslateProvider>
  );
}
```

## 6. Usa links y traducciones consciente de la localización

Crea `src/components/header.tsx`:

```tsx
import { useTranslations } from "@better-translate/react";

import { translator } from "../lib/i18n";
import { I18nLink, useLocale } from "../lib/navigation";

export function Header() {
  const currentLocale = useLocale();
  const { t } = useTranslations<typeof translator>();

  return (
    <header>
      <h1>{t("home.title")}</h1>
      <I18nLink to="/" params={{ lang: currentLocale }}>
        Home
      </I18nLink>
    </header>
  );
}
```

## TanStack Start

Si estás usando TanStack Start, mantén la misma configuración de enrutamiento y navegación. Luego agrega los helpers del servidor de `@better-translate/tanstack-router/server` para derivar la configuración por solicitud, fijar el locale activo al inicio y leer traducciones dentro de loaders o funciones del servidor.

```ts
import { configureTranslations } from "@better-translate/core";
import {
  createServerHelpers,
  getRequestConfig,
  setRequestLocale,
} from "@better-translate/tanstack-router/server";
import { createServerFn } from "@tanstack/react-start";
import { getRequest } from "@tanstack/react-start/server";

import { en } from "./messages/en";
import { es } from "./messages/es";
import { routing, type AppLocale } from "./routing";

export const requestConfig = getRequestConfig(async () => {
  const request = getRequest();
  const pathname = new URL(request.url).pathname;
  const locale = pathname.startsWith("/es") ? "es" : routing.defaultLocale;

  return {
    locale,
    translator: await configureTranslations({
      availableLocales: routing.locales,
      defaultLocale: routing.defaultLocale,
      fallbackLocale: routing.defaultLocale,
      messages: { en, es },
    }),
  };
});

const { getTranslations } = createServerHelpers(requestConfig);

export const getHomeCopy = createServerFn({ method: "GET" })
  .handler(async () => {
    const locale = (await requestConfig()).locale as AppLocale;

    setRequestLocale(locale);
    const t = await getTranslations({ locale });

    return {
      title: t("home.title"),
    };
  });
```
