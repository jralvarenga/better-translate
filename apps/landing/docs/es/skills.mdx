# Skills

La carpeta `skills/` en
el repositorio es una guía de aprendizaje escrita en lenguaje simple. Explica cómo funciona la librería
sin necesidad de leer el código fuente completo primero.

## Orden de lectura

Si eres nuevo, lee los archivos de skills en este orden:

1. `skills/core/README.md` — El motor de traducciones: claves tipadas, parámetros, fallback, carga asíncrona y helpers globales.
2. `skills/react/README.md` — El adaptador React: BetterTranslateProvider y useTranslations().
3. `skills/expo/README.md` — Expo y React Native usando el adaptador React existente.
4. `skills/nextjs/README.md` — El adaptador Next.js: enrutamiento de locales, helpers de servidor y wrappers de navegación.
5. `skills/combined/README.md` — Cómo core, React y Next.js trabajan juntos en una app real.
6. `skills/examples/README.md` — Un mapa de las apps de ejemplo — copia la que se adapte a tu arquitectura.
7. `skills/workspace/README.md` — Cómo está organizado el monorepo: paquetes vs apps, scripts de compilación y archivos de config.

## Modelo mental simple

Better Translate tiene una idea principal: **configurar las traducciones una vez, usar helpers tipados en todas partes.**

El paquete core gestiona la lógica de traducción.

El paquete React hace que esa lógica sea fácil de usar en componentes cliente.
Expo y React Native usan ese mismo paquete React en lugar de un adaptador nativo separado.

El paquete Next.js hace que esa lógica sea fácil de usar con enrutamiento, helpers de servidor y navegación con locale.

## ¿Qué guía debería leer?

- → ¿Solo necesitas traducciones en TypeScript o en un servidor? Lee `skills/core/README.md`
- → ¿Construyendo una app React? Lee `skills/react/README.md`
- → ¿Construyendo con Expo o React Native? Lee `skills/expo/README.md`
- → ¿Construyendo una app Next.js App Router? Lee `skills/nextjs/README.md`
- → ¿La app tiene partes de servidor y cliente? Lee `skills/combined/README.md`
- → ¿Aprendes mejor copiando proyectos funcionales? Lee `skills/examples/README.md`
