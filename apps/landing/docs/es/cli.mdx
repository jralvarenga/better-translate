# CLI

Usa `@better-translate/cli` cuando quieras que Better Translate cree y actualice tus archivos de locale.

No necesitas la CLI para usar los paquetes de runtime. Es opcional.

## 1. Instala los paquetes

```sh
npm install -D @better-translate/cli @ai-sdk/openai
```

O instala otro paquete de provider:

```sh
npm install -D @better-translate/cli @ai-sdk/anthropic
# o
npm install -D @better-translate/cli @ai-sdk/moonshotai
```

La CLI ya no incluye helpers de provider. Tu app instala y configura el provider de AI SDK directamente.

## 2. Crea un archivo de locale fuente

Crea `src/messages/en.json`:

```json
{
  "home": {
    "title": "Hello",
    "description": "Welcome to the app"
  }
}
```

## 3. Crea la configuración de la CLI

Crea `better-translate.config.ts`. El flujo con `model` directo es el mismo para cualquier provider:

```ts
import { openai } from "@ai-sdk/openai";
import { defineConfig } from "@better-translate/cli/config";

export default defineConfig({
  sourceLocale: "en",
  locales: ["es", "fr"],
  model: openai("gpt-5"),
  messages: {
    entry: "./src/messages/en.json",
  },
});
```

```ts
import { anthropic } from "@ai-sdk/anthropic";
import { defineConfig } from "@better-translate/cli/config";

export default defineConfig({
  sourceLocale: "en",
  locales: ["es", "fr"],
  model: anthropic("claude-sonnet-4-5"),
  messages: {
    entry: "./src/messages/en.json",
  },
});
```

```ts
import { moonshotai } from "@ai-sdk/moonshotai";
import { defineConfig } from "@better-translate/cli/config";

export default defineConfig({
  sourceLocale: "en",
  locales: ["es", "fr"],
  model: moonshotai("kimi-k2"),
  messages: {
    entry: "./src/messages/en.json",
  },
});
```

Si necesitas configuración personalizada del provider, crea el modelo tú mismo y pásalo a la CLI. Las credenciales y la configuración específica viven en el paquete del provider que elijas:

```ts
import { createOpenAI } from "@ai-sdk/openai";
import { defineConfig } from "@better-translate/cli/config";

const model = createOpenAI({
  apiKey: process.env.OPENAI_API_KEY,
  baseURL: process.env.OPENAI_BASE_URL,
})("gpt-5");

export default defineConfig({
  sourceLocale: "en",
  locales: ["es", "fr"],
  model,
  messages: {
    entry: "./src/messages/en.json",
  },
});
```

Este es un cambio incompatible con versiones anteriores de la CLI que exportaban `openai(...)` desde `@better-translate/cli/config`.

## 4. Marca strings pendientes

Marca strings con `{ bt: true }` para escribir el texto fuente directamente. La CLI generará las claves por ti:

```ts
import { t } from "@better-translate/core";

export function navLabel() {
  return t("Home", { bt: true });
}
```

`{ bt: true }` flexibiliza el primer argumento para que puedas escribir el string fuente directamente. En runtime devuelve el string sin cambios hasta que ocurra la extracción.

## 5. Extrae claves fuente

```sh
npx bt extract --config ./better-translate.config.ts
```

Ese comando busca llamadas marcadas `t(..., { bt: true })`, agrega las claves faltantes a tu archivo de locale fuente y reescribe las llamadas exitosas a `t("namespace.generatedKey")`.

## 6. Ejecuta el generador

```sh
npx bt generate --config ./better-translate.config.ts
```

Ese comando crea los archivos de locale destino junto a tu archivo fuente.

## 7. Usa los archivos generados en tu app

Después de que existan los archivos, impórtalos en tu configuración de `@better-translate/core` igual que cualquier otro archivo de locale escrito a mano.

## Markdown

Si también quieres generar markdown localizado, agrega la opción `markdown.rootDir`. Esto funciona igual con cualquier modelo de provider de AI SDK:

```ts
import { openai } from "@ai-sdk/openai";
import { defineConfig } from "@better-translate/cli/config";

export default defineConfig({
  sourceLocale: "en",
  locales: ["es", "fr"],
  model: openai("gpt-5"),
  messages: {
    entry: "./src/messages/en.json",
  },
  markdown: {
    rootDir: "./content/docs",
  },
});
```
