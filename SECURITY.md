# Security Policy

## Reporting a Vulnerability

Please do not report security vulnerabilities through public GitHub issues, pull requests, or discussions.

If you believe you have found a security issue in this repository, use GitHub's private vulnerability reporting flow:

- Open the repository Security tab
- Choose `Report a vulnerability`
- Submit the details privately through GitHub Security Advisories

If you prefer a direct link, use:

https://github.com/jralvarenga/better-translate/security/advisories/new

Please include:

- a clear description of the issue
- steps to reproduce or a proof of concept
- the affected package or area of the repository
- any suggested remediation or relevant context

## What to Expect

- We will review reports privately
- We may ask follow-up questions to confirm impact and scope
- We will work on a fix and coordinate disclosure when appropriate
- We ask that you give us reasonable time to investigate and ship a fix before public disclosure

## Disclosure Guidance

To help protect users, please avoid sharing exploit details publicly until a fix or mitigation is available.

Once a fix is ready, maintainers may publish an advisory or release notes describing the impact, affected versions, and remediation steps.
